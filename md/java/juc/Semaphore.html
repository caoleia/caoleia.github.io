# JUC工具类: Semaphore详解

  • 使用示例
  • 类层次结构
  • 实现原理

Semaphore(信号量)底层是基于AbstractQueuedSynchronizer来实现的。Semaphore称为信号量，提供了资源的并发访问数量控制，它允许n个任务同时访问某个资源，可以将信号量看做是在向外分发使用资源的许可证，只有成功获取许可证，才能使用资源。

# 使用示例

 //初始有10个共享资源。第二个参数是恐怕或非公平选项
 Semaphore semaphore = new Semaphore(10,true);
 //每次获取一个，如果获取不到，则线程阻塞
 semaphore.acquire();  
 //释放资源
 semaphore.release();

# 实现原理

与锁相似，acquire与release分别调用内部类sync实现的AQS模版方法acquireSharedInterruptibly与releaseShared。

通过名字可以发现，获取锁的操作是可以共享的。

主要思路是在acquire里对state进行CAS减操作，减到0之后在调用acquire线程阻塞; 在release里对state进行CAS加操作，从0到1时需要唤醒线程。

public void acquire() throws InterruptedException {
    sync.acquireSharedInterruptibly(1);
}

public void release() {
    sync.releaseShared(1);
}
