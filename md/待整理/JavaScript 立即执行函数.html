# 语法解释

最初我以为是一个括号包裹匿名函数，再在后面加个括号调用函数，最后达到函数定义后立即执行的目的，后来发现加括号的原因并非如此。 通常我们声明一个函数有以下几种方式：

// 声明函数f1
function f1() {
 console.log("f1");
}
// 通过()来调用此函数
f1();
 
//一个匿名函数的函数表达式，被赋值给变量f2:
var f2 = function() {
 console.log("f2");
}
//通过()来调用此函数
f2();
 
//一个命名为f3的函数的函数表达式(这里的函数名可以随意命名，可以不必和变量f3重名)，被赋值给变量f3:
var f3 = function f3() {
 console.log("f3");
}
//通过()来调用此函数
f3();

我们以f2为例，在匿名函数赋值给f2之后，我们可以通过f2()来调用方法。 我们发现这里f2只是这个匿名函数的一个引用变量，那么既然f2()能够调用这个函数，我把f2替换成函数本身可以吗?

运行之后得到如下结果:Uncaught SyntaxError: Unexpected token (

产生这个错误的原因是，Javascript引擎看到function关键字之后，认为后面跟的是函数声明语句（f1），不应该以圆括号结尾。 解决方法就是让引擎知道，我们要执行的是一个函数表达式(f2)。 那么用括号包裹匿名函数的作用就出来了：让Javascript引擎知道()内是一个函数表达式。 **(function(){...})();**就可以解释为声明了一个匿名函数，并且立即执行它。

# 实际应用

# 1.避免污染全局变量

首先这个立即执行函数不需要命名，避免了函数名的污染。如果你的代码在页面代码加载完成之后，不得不执行一些设置工作，所有的这些工作只需要执行一次，所以没有理由创建一个可复用的命名的函数。 其次JavaScript没有块级作用域，是函数作用域，这样在IIFE内部形成了一个单独的作用域，jquery插件一般以这样的形式出现：

(function($){
  //插件实现代码
})(jQuery);

3

这里的jquery其实是该匿名函数的参数，联想一下我们调用匿名函数时候是用f2()那么匿名带参数的就是f(2args),这里把jquery作为参数传入该函数，那么在函数内部使用形参$的时候就不会影响到外部环境，因为有些插件也会用到$这个限定符。 这样在IIFE内部，我们可以放心使用传入的参数，任意声明变量、方法而不用担心与其他JavaScript代码有冲突，你在这个函数内部可以随意折腾。

# 2.封装私有变量

看一段《JavaScript设计模式与开发实践》中封装数据的代码：

var myObject = (function(){
    var __name = 'sven'; 	 // 私有（private）变量
    return {
        getName: function(){ // 公开（public）方法
        return __name;
        }
    }
})();
console.log( myObject.getName() ); // 输出：sven
console.log( myObject.__name ) // 输出：undefined
