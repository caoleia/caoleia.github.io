# 比较器复合

一般函数式接口都用作方法的参数，我们只是使用Lambda表达式来简化这种写法

函数式接口和Lambda表达式还可用作方法的返回值(像JS一样，返回值可以是一个function)，传递代码的回调，将这两种用法结合起来，可以构造复合的函数，使程序简洁易读。 以数组的排序方法为例：

//一个简单的类，我们要以年龄age排序
public class People {
    public Integer getAge() {
        return age;
    }

    public void setAge(Integer age) {
        this.age = age;
    }

    private Integer age;

    public People(Integer age){
        this.age = age;
    }
}

People[] peoples = {
        new People(18),
        new People(10),
        new People(20)
};

//普通方式
 Arrays.sort(peoples, new Comparator<People>() {
    @Override
    public int compare(People o1, People o2) {
        return o1.getAge().compareTo(o2.getAge());
    }
});

//lambda表达式方式
Arrays.sort(peoples,(p1,p2)->p1.getAge().compareTo(p2.getAge()));
//使用Comparator接口的静态方法comparing
Arrays.sort(peoples, Comparator.comparing(p->p.getAge()));
//继续将p->p.getAge改为方法引用
Arrays.sort(peoples, Comparator.comparing(People::getAge));

看一下comparing的源码

public static <T, U extends Comparable<? super U>> Comparator<T> comparing(
        Function<? super T, ? extends U> keyExtractor)
{
    Objects.requireNonNull(keyExtractor);
    return (Comparator<T> & Serializable)
        (c1, c2) -> keyExtractor.apply(c1).compareTo(keyExtractor.apply(c2));
}

comparing方法构建并返回了一个符合Comparator接口的Lambda表达式。 在这个Lambda表达式中，有两个参数c1,c2，分别执行了keyExtractor.apply(c1)与keyExtractor.apply(c2)。

感觉定义这种方法的目的就是因为这种场景会经常出现，所以官方定义了一个这种比较方法， 但是对于不熟悉的人来说代码不好理解，不如直接使用lambda表达式方式直观，必须点进comparing方法的源码看看才能够明白。 其实最终的目的并不是这么简单，Comparator还有很多其它类似返回函数式接口的例子，熟悉后这种形式可以更好的简略代码的写法。

default Comparator<T> reversed() {
    return Collections.reverseOrder(this);
}

default Comparator<T> thenComparing(Comparator<? super T> other) {
    Objects.requireNonNull(other);
    return (Comparator<T> & Serializable) (c1, c2) -> {
        int res = compare(c1, c2);
        return (res != 0) ? res : other.compare(c1, c2);
    };
}

看一个使用的例子，将学生列表按照分数倒序排(高分在前)，分数一样的，按照名字进行排序，代码如下所示：

students.sort(Comparator.comparing(Student::getScore)
                .reversed()
                .thenComparing(Student::getName));

# 函数复合(Function)

Function<T,R> ,R apply(T t) 函数转换，输入类型T，返回类型R

和Comparator类似，Function接口中也有两个复合方法andThen和compose。

//aftert函数转换，再进行一次函数转换
default <V> Function<T, V> andThen(Function<? super R, ? extends V> after) {
    Objects.requireNonNull(after);
    return (T t) -> after.apply(apply(t));
}
//before函数转换，进行一次函数转换
default <V> Function<V, R> compose(Function<? super V, ? extends T> before) {
    Objects.requireNonNull(before);
    return (V v) -> apply(before.apply(v));
}

Function<Integer,Integer> f=x->x+1;
Function<Integer,Integer> g=x->x*2;
Function<Integer,Integer> h=f.andThen(g);
int result=h.apply(1);//4

# 谓词复合(Predicate)

Predicate, boolean test(T t) 谓词，测试输入是否满足条件

词接口包括三个方法：negate、and和or，让你可以重用已有的Predicate来创建更复杂的谓词。比如，你可以使用negate方法来返回一个Predicate的非，比如苹果不是红的。

Predicate<Apple> redApple = a->"red".equals(a.getColor());
//非
Predicate<Apple> notRedApple=redApple.negate();
//与 把两个Lambda用and方法组合起来
Predicate<Apple> redAndHeavyApple=redApple.and(a->a.getWeight()>50);
//或
Predicate<Apple> redAndHeavyAppleOrGreen=redApple.and(a->a.getWeight()>150)
    .or(a->"green".equals(a.getColor()));
