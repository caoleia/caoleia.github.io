总结JavaScript判断数据类型的各种方法，各种情况下的最佳实践。

# typeof

在ES5中有6种数据类型,typeof把类型信息当作字符串返回，返回值有"number," "string," "boolean," "object," "function," "undefined" 语法：

if(typeof a!="undefined"){}
1

对于object、Array,Null，typeof一律返回object，所以它的适用场景也很明显了：

  1. 判断数据类型，适用于只需要明确到6种数据类型的场景，无法区分object、Array,Null
  2. 确定变量是否存在，typeof a!="undefined",存在的意思就是不管他是什么值，null也是存在只不过没有值

# instanceof

instanceof运算符用来判断一个构造函数的prototype属性所指向的对象是否存在另外一个要检测对象的原型链上。

说的通俗一点就是判断一个变量是否是某个对象的实例，它的机制比较难懂，需要非常了解JavaScript的原型知识。 在确保不会错的前提下，我主要使用它完成：

  1. 判断一个变量是否是某个对象或其子对象的实例
  2. 作为type of判断object的替代方法，可以明确判断是否Array

需要注意以下问题

var a=new Array();
alert(a instanceof Array);  //true
alert(a instanceof Object); //true

# Object.prototype.toString.call

终极解决方案 工作中还没遇到过有这样写的，今天在书上第一次发现这种用法，并且所有类型的变量都可以检测出来。

先看一下结果：

console.log(
    Object.prototype.toString.call(num),    // '[object Number]'
    Object.prototype.toString.call(str),    // '[object String]'
    Object.prototype.toString.call(bool),   // '[object Boolean]'
    Object.prototype.toString.call(arr),    // '[object Array]'
    Object.prototype.toString.call(obj),    // '[object Object]'
    Object.prototype.toString.call(func),   // '[object Function]'
    Object.prototype.toString.call(und),    // '[object Undefined]'
    Object.prototype.toString.call(nul),    // '[object Null]'
    Object.prototype.toString.call(date),   // '[object Date]'
    Object.prototype.toString.call(reg),    // '[object RegExp]' 
    Object.prototype.toString.call(error)   // '[object Error]'
);

所有类型都细致的检测出来了，甚至连RegExp、Error对象都可以检测。 这里对于Number、String、Boolean无论是var a = 'xx'，还是 var a = new String('xx')方式创建的，都会返回我们想要的结果。 这里只有 "纯粹" 的object对象才会返回Object。 唯一的缺点就是不具备instanceof可以区分自定义对象类型的功能了。

**Object.prototype.toString.call:纯粹、精确的判断数据类型

原理： 每个对象如果没有重写toString()方法，都会调用原型链上的Object.prototype.toString方法。 正常它会返回"[object type]"，通过Object.prototype.toString.call就可以保证调用未重写的toString方法。
